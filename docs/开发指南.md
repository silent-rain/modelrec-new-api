# 开发指南

## 环境变量

```sh
cp .env.example .env


# PORT=3000
# SQL_DSN=postgresql://user:123456@postgres:localhost:5432/new-api  # 如使用PostgreSQL，取消注释并修改
# REDIS_CONN_STRING=redis://localhost:6379         # 如使用Redis，取消注释并修改
```

## PostgreSQL

```sql
-- 创建用户
CREATE USER newapi WITH PASSWORD 'pass';


-- 创建数据库
CREATE DATABASE "new-api" OWNER newapi;
```

配置 PostgreSQL 允许外部连接:

```sh
# 修改 pg_hba.conf
sudo vim /etc/postgresql/18/main/pg_hba.conf
# 在文件末尾添加:
host    all    all    172.17.0.0/16    md5

# 修改 postgresql.conf
sudo vim /etc/postgresql/18/main/postgresql.conf
# 修改: listen_addresses = '*'

# 重启
sudo systemctl restart postgresql
```

## Redis

配置 Redis 允许外部连接:

```sh
# 修改 redis.conf
sudo vim /etc/redis/redis.conf
# 修改: bind 0.0.0.0
# 建议设置密码: requirepass your_strong_password

# 重启
sudo systemctl restart redis
```

验证步骤:

```sh
# 1. 重新构建并启动容器
docker-compose down
docker-compose up -d --build

# 2. 查看日志确认数据库连接成功
docker logs new-api -f

# 3. 如果连接失败，测试网络连通性
docker exec -it new-api ping host.docker.internal

```

## 开发模式

```sh
# 前端
cd web/default
bun install   # 使用 bun 安装新版前端依赖

bun run dev # 使用 bun 运行新版开发服务器


# 后端
# go run main.go

go run main.go --log-dir ./logs
```

## 发布模式

```sh
# 前端
cd web/default
bun run build


# 后端
go build -o modelrec
./modelrec
```
